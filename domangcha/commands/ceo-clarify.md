# /ceo-clarify — 입력 정제 후 CEO 자동 실행

사용자의 입력을 받아 CEO가 최적으로 이해할 수 있는 형태로 정제한 뒤,
자동으로 `/ceo`를 실행합니다.

모호하든, 복잡하든, 언어가 섞였든 — 모든 입력을 처리합니다.

---

## 실행 순서

### STEP 1: 입력 분석

원본 입력($ARGUMENTS)을 분석:
- 핵심 목표가 명확한가?
- 기술 스택 언급이 있는가?
- 범위가 파악 가능한가?
- 완료 기준을 추론할 수 있는가?

### STEP 2: 보완 질문 (필요 시만, 최대 3개)

핵심 정보가 빠진 경우만 질문. 한 번에 하나씩.

질문 우선순위:
1. 목표가 불분명할 때 — "어떤 결과를 원하세요?"
2. 범위가 너무 넓을 때 — "이번에 꼭 포함해야 할 것 vs 나중에 해도 될 것은?"
3. 기술 맥락이 없을 때 — "기존 코드베이스가 있나요? 선호하는 스택은?"

충분한 정보가 있으면 질문 없이 바로 STEP 3으로.

### STEP 3: 정제 결과 출력

```
[INTENT PARSED] 🧠
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
원본 입력:
  "<사용자가 입력한 그대로>"

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
정제된 태스크:
  "<구조화되고 명확한 실행 명세>"

목표:   <핵심 목표 1줄>
범위:   <포함할 것 / 제외할 것>
전제:   <파악된 스택, 제약, 맥락>
기준:   <완료를 판단하는 조건>
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
→ /ceo 실행 중...
```

### STEP 4: /ceo 자동 실행

정제된 태스크를 기준으로 `/ceo` 파이프라인 즉시 시작.
(SIZE ASSESSMENT → PHASE 0 → FULL PIPELINE)

---

## 사용 예시

```
/ceo-clarify "로그인 만들어"
/ceo-clarify "우리 앱 느린데 고쳐줘"
/ceo-clarify "저번에 말한 그 기능 있자나 그거"
/ceo-clarify "결제도 되고 관리자도 있고 유저도 관리하는거"
```
