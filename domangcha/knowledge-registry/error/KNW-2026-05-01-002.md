## Summary
소스코드에 시크릿 하드코딩 — 즉시 차단 + 로테이션 필수

## Context
DC-DEV-* 에이전트가 API_KEY, password, secret 등을 소스 파일에 직접 삽입하는 경우.
.env 사용 방법을 모르거나 빠른 구현을 위해 임시로 하드코딩 후 잊어버리는 패턴.

## Rule
- GATE 1 + DC-SEC 이중 차단
- 발견 즉시: 소스에서 삭제 → .env + 환경변수로 교체 → 시크릿 로테이션
- .gitignore에 .env 반드시 포함 확인
- 관련: ERR-002

## Example
```
# 위반
const API_KEY = "<REDACTED-EXAMPLE-KEY>"

# 처방
const API_KEY = process.env.OPENAI_API_KEY
// .env: OPENAI_API_KEY=<your-actual-key-here>
// .gitignore: .env
```
